# Техзадание: Gina Decor — простой inventory tracker

ТЗ для Claude Code. Никакой бухгалтерии — обычный складской учёт.

Текущее состояние: репозиторий `indisbelief/gina-decor`, задеплоен на Vercel
(`gina-decor.vercel.app`). Сейчас один статический `index.html`, данные
в IndexedDB на устройстве.

**Задача:** переделать в приложение с общей базой на сервере, чтобы двое
(владелец и мама) видели одни и те же данные с разных телефонов.

---

## 1. Что должно уметь

1. Список товаров с фото, поиском и фильтрами.
2. Отметить «продано» / вернуть в наличие — в один тап.
3. Несколько фото на товар, съёмка прямо с телефона.
4. Добавить / отредактировать / архивировать товар.
5. Общая база: изменил один — видит другой.
6. Выгрузка в CSV.
7. Вешается на домашний экран телефона как приложение (PWA).

---

## 2. Модель данных

Postgres. Суммы — `numeric(10,2)`.

### `items`

| Поле | Тип | Обязательное | Комментарий |
|---|---|---|---|
| `id` | uuid pk | — | |
| `sku` | text unique | авто | напр. `GD-0041` |
| `merk` | text | **да** | бренд |
| `model` | text | нет | коллекция / модель |
| `soort` | text | нет | тип: сервиз, ваза, столовые приборы |
| `aantal_delen` | int | нет | размер сервиза |
| `staat` | enum | нет | `nieuw` / `als_nieuw` / `gebruikt` |
| `locatie` | text | нет | Section 1, Home, website… |
| `inkoopprijs` | numeric | нет | за сколько куплено |
| `vraagprijs` | numeric | нет | за сколько выставлено |
| `verkoopprijs` | numeric | нет | за сколько реально продано |
| `inkoopdatum` | date | нет | необязательное, но полезное на будущее |
| `verkoopdatum` | date | нет | ставится автоматом при отметке «продано» |
| `leverancier` | text | нет | у кого куплено, простой текст |
| `status` | enum | да | `voorraad` / `verkocht` / `gereserveerd` |
| `notities` | text | нет | |
| `created_at`, `updated_at` | timestamptz | — | |
| `archived_at` | timestamptz null | — | soft delete, не удалять физически |

Обязательное поле только одно — `merk`. Всё остальное можно оставить пустым
и дозаполнить потом. Не блокировать сохранение из-за пустых полей.

### `photos`

`id` · `item_id` fk · `url` · `volgorde` · `is_hoofdfoto` · `created_at`

Файлы — в Vercel Blob, в базе только URL. Сжимать на клиенте перед загрузкой
(максимум ~1600px по длинной стороне, jpeg, качество ~0.6).

## 3. Экраны

**Список (главный)**
- Карточка: миниатюра фото, бренд + модель, тип · место · цена,
  крупная кнопка «продано» справа.
- Сверху счётчики: сколько в наличии и на какую сумму, сколько продано.
- Поиск по бренду / модели / типу / месту.
- Фильтры: статус (все / в наличии / продано), место, бренд.
- Сортировка: по дате добавления, по цене, по бренду.
- Приоритет — мобильный вид, пользоваться одной рукой.

**Карточка товара**
- Галерея фото, кнопка «добавить фото» (камера или галерея).
- Все поля редактируются на месте, сохранение автоматом при потере фокуса.
- Кнопка «Продано» → спрашивает цену продажи (необязательно, можно пропустить)
  и ставит дату.
- Архивировать (не удалять).

**Добавить товар**
- Минимальная форма: бренд обязателен, остальное по желанию.
- Сразу после создания предложить сделать фото.

**Настройки**
- Выгрузка CSV.
- Список мест (locaties) для быстрого выбора.

## 4. Технически

- **Next.js (App Router) + TypeScript** — Vercel уже подключён.
- **База:** Vercel Postgres (Neon), схема через Drizzle или Prisma + миграции.
- **Фото:** Vercel Blob.
- **Доступ:** двое пользователей, ничего сложного не надо.
  Простой вариант — один общий пароль в переменной окружения,
  либо NextAuth с Google. На выбор, что проще поднять.
- **PWA:** manifest + иконка, чтобы вешалось на домашний экран.
  Офлайн — желательно хотя бы чтение списка.
- Никаких float для денег — decimal или центы в integer.

---

## 5. Миграция текущих данных

В `index.html` есть константа `SEED` — 73 позиции. Перенести в `items`:

`brand → merk` · `model → model` · `piece → soort` · `setSize → aantal_delen` ·
`location → locatie` · `price → inkoopprijs` · `condition → staat` ·
`sold → status` (`true → verkocht`, `false → voorraad`)

Старый `index.html` после миграции убрать из корня, чтобы не путался.

---

## 6. Что НЕ делать сейчас

- Никакой бухгалтерии: BTW, маржа, налоговые отчёты — не нужны.
- Никакой автосинхронизации с Shopify. Позже — импорт CSV с ручным
  подтверждением, но не в первой версии.
- Роли и права, мультисклад, штрихкоды, ценники — не нужны.
